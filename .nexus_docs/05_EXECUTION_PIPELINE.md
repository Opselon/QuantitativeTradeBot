# Layered Execution Pipeline Design

## 1. Overview
The Nexus Trading Engine (NTE) coordinates order routing through a formal layered execution pipeline, rather than ad hoc service calls. This enforces strict safety controls, traceabilities, and consistent transactional boundaries.

```
+---------------+
|  TradeSignal  | (Generated by Hosted Strategy)
+-------+-------+
        |
        v
+-------+-------+
|  SignalRouter | (Emits Signal to Coordinator)
+-------+-------+
        |
        v
+-------+-------+
|  OrderIntent  | (Constructed and basic validations applied)
+-------+-------+
        |
        v
+-------+----------------+
| PreTradeRiskEvaluator  | (Runs IRiskManager rules)
+-------+----------------+
        |
        +-- (REJECTED) -> Persist rejected order & notify audit
        |
        +-- (PASSED)   -> Save pending order & progress
        |
        v
+-------+---------------+
|  IExecutionGateway    | (Dispatches command to MT5 bridge)
+-------+---------------+
        |
        v
+-------+---------------+
|  ExecutionReport      | (Saves filled order, updates position / account)
+---------------+-------+
```

## 2. Key Components
- **`PipelineContext`**: Carries diagnostic metadata including a unique `CorrelationId` and transaction timestamps. This context flows across all layers of the pipeline.
- **`TradeSignal`**: Generated by a strategy, specifying intention (Direction, Volume, Target Price, StopLoss, and TakeProfit).
- **`OrderIntentFactory`**: Maps the signal parameters into a formalized system-wide `OrderIntent`.
- **`PreTradeRiskEvaluator`**: Retrieves account parameters and executes active risk rules (`IRiskManager`) before any broker-facing actions occur.
- **`IExecutionGateway`**: The decoupled port leading to the MT5 bridge adaptor.
- **`ExecutionAuditService`**: Logs structured diagnostic markers for each phase, creating an immutable audit trail.
