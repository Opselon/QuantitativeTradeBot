## 🏛️ Nexus Trading Engine (NTE) Architecture Summary
**Style:** Decoupled Hexagonal / Clean Architecture

- **Nexus.Core:** Zero external dependencies. Contains `Symbol`, `Money`, `Tick`, `Bar`, `IStrategy`. Uses *Zero-Allocation Tick Path* (Readonly Structs, Ring Buffers).
- **Nexus.Application:** Implements execution logic, `IExecutionGateway`, `ExecutionCoordinator`, `IMt5TradingService` (Simulated vs Real Routing), Security Hardening.
- **Nexus.Infrastructure:** Adapters (EF Core PostgreSQL, ADO.NET binary copy, System.Threading.Channels for Background Workers).
- **Native C++:** High-performance math (EMA rolling) via P/Invoke to bypass JIT.
- **Nexus.WpfUi:** Desktop UI layer (Deferred).
